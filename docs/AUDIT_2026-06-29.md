## Findings addressed

### 1. Per-account auth / client cache

**Problem.** `widgets/clients.py` / `widgets/graph_helper.py` rebuilt a fresh
`GraphAuth` + `GraphClient` on every view call. MSAL token-cache reload and
`PublicClientApplication` construction happened repeatedly.

**Fix.**
- `CloudyApplication` now owns a thread-safe `_account_clients` cache with
  `get_account_client()`, `set_account_client()`, and `evict_account_client()`.
- `widgets/clients.build_account_client()` reuses cached clients per account id.
- `window._sign_out()` and `_remove_account()` evict the cached client so stale
  tokens are never reused after sign-out.

**Files.** `src/cloudy/application.py`, `src/cloudy/widgets/clients.py`,
`src/cloudy/window.py`, `tests/unit/test_clients.py`.

### 2. Microsoft 365 `create_share_link` path → drive/item mismatch

**Problem.** `OneDriveFiles.create_share_link()` delegated to an unimplemented
`AbrauneggClient` stub, treating a local path as if it were a drive item id.

**Fix.**
- `OneDriveFiles._resolve_path()` maps a local filesystem path back to the
  correct `(drive_id, relative_path)` using remembered mount records (now stored
  with their mountpoint).
- `GraphClient.item_by_path()` resolves `/drives/{id}/root:/{path}` (or
  `/me/drive/root:/{path}`) to a driveItem.
- `GraphClient.create_share_link()` then requests the share link for the
  resolved item.
- Mount records now include the `mountpoint` field so SharePoint/team-drive
  paths are attributed to the right drive.

**Files.** `src/cloudy/modules/microsoft365/files.py`,
`src/cloudy/modules/microsoft365/graph.py`,
`src/cloudy/modules/microsoft365/mounts.py`,
`src/cloudy/widgets/files_view.py`, `tests/unit/test_files.py`.

### 3. Graph pagination for non-calendar collections

**Problem.** Calendar views were already paginated, but OneNote notebook/section/
page endpoints used a single `_get()` call and could silently truncate.

**Fix.** `list_notebooks()`, `list_note_sections()`, and `list_note_pages()` now
use `_get_all()` and follow `@odata.nextLink`.

Mail folders, drives, sites, groups, team channels, and contacts were already
using `_get_all()`.

**Files.** `src/cloudy/modules/microsoft365/graph.py`.

### 4. Whole-UI rebuild churn

**Problem.** Mail and chat lists were cleared and fully rebuilt on every data
load, destroying widgets, selection, and scroll position.

**Fix.**
- Added `patch_listbox()` to `widgets/source_nav.py`: a generic helper that
  reuses existing rows, updates them in place, removes rows for deleted items,
  appends new rows, and restores selection.
- `MailView._render()` now uses `patch_listbox()`; `_mail_row()` was split into
  a create path and an `_update_mail_row()` update path so labels, read-state,
  and flags refresh without rebuilding the widget tree.
- `ChatView._render_filtered()` uses `patch_listbox()`; `_chat_row()` reuses the
  row widget while rebuilding its content.

**Files.** `src/cloudy/widgets/source_nav.py`,
`src/cloudy/widgets/mail_view.py`, `src/cloudy/widgets/chat_view.py`.

### 5. Naive iCalendar parser

**Problem.** `core/ics.py` split parameters on `;` and values on `:` without
regard for RFC 5545 escaping or quoted parameter values, so escaped commas,
newlines, or semicolons in names/CNs broke parsing.

**Fix.**
- `_unescape()` reverses `\;`, `\,`, `\\`, and `\n`/ `\N` → newline.
- `_split()` finds the first *unescaped* colon and splits parameters with
  quote-aware semicolon handling.
- `_split_params()` preserves quoted values that contain semicolons.
- `build_reply()` now escapes newlines as well.

**Files.** `src/cloudy/core/ics.py`, `tests/unit/test_ics.py`.

### 6. Google OAuth redirect receiver cleanup

**Problem.** The local HTTP server used `handle_request()` in a daemon thread
and only called `server_close()`, leaving the accept socket and thread in an
uncertain state on timeout or error.

**Fix.**
- `GoogleAuth.sign_in_interactive()` now runs `serve_forever()` in a daemon
  thread and always calls `server.shutdown()` + `server.server_close()` in a
  `finally` block, joining the thread with a short timeout.
- `server.allow_reuse_address = True` avoids transient bind failures.
- State mismatch is still checked before exchanging the code.

**Files.** `src/cloudy/core/auth/google_oauth.py`.

### 7. Provisioned binary trust model

**Problem.** `ensure_rclone()` downloaded the latest version and verified it
against a SHA256SUMS file fetched from the same origin. A compromise of
`downloads.rclone.org` could serve a matching malicious checksum.

**Fix.**
- Pinned `RCLONE_VERSION = "v1.74.3"` and hard-coded SHA-256 sums for the
  `linux-amd64` and `linux-arm64` zip archives.
- `ensure_rclone()` no longer follows `version.txt`; it downloads only the
  pinned version and rejects the archive if the checksum does not match.
- The extracted binary is chmodded to `0o755` (owner-writable, group/other
  read+execute only).

**Files.** `src/cloudy/core/provisioner.py`,
`tests/unit/test_provisioner.py`.

## Remaining / accepted risks

1. **File browser grid/list still rebuilds fully on every sort/filter/view
   switch.** The new `patch_listbox()` helper could be extended to a
   `patch_flowbox()` / tree-aware list patcher for this surface, but that is a
   larger refactor than the current pass.
2. **Rclone provisioning still trusts the TLS certificate of
   `downloads.rclone.org`** for availability. Integrity is now pinned, but a
   future audit may want GPG signature verification of the checksum file or
   distribution through the Flatpak runtime only.
3. **Google OAuth loopback redirect** uses a random localhost port. This is the
   standard installed-app PKCE flow, but the HTML response page is still a
   simple string rather than a CSP-hardened document.
4. **MSAL token cache** remains encrypted only by the OS keyring/libsecret
   boundary; the per-account client cache in `CloudyApplication` holds an
   in-memory `GraphAuth` object, which is expected and bounded by the number of
   configured accounts.
